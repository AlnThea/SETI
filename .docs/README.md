## 📋 Documentation Standards

### Updating Documentation

Semua dokumentasi harus:
- ✅ Ditulis dalam Markdown format
- ✅ Menggunakan bahasa yang jelas dan mudah dipahami
- ✅ Include contoh code jika relevan
- ✅ Update "Last Updated" date di footer
- ✅ Follow struktur yang konsisten

### When to Update

- **PLANNING.md** - Saat ada perubahan roadmap atau timeline
- **TASKS.md** - Daily, setelah menyelesaikan task
- **DEVLOG.md** - Daily, di akhir hari kerja
- **DOCUMENTATION.md** - Saat ada perubahan architecture atau API
- **CODE_STANDARDS.md** - Saat ada best practice baru
- **PROJECT_STRUCTURE.md** - Saat ada perubahan struktur folder

## 📝 Contributing to Documentation

### How to Contribute

1. **Identify Gap** - Find missing or outdated information
2. **Create Branch** - `git checkout -b docs/update-xyz`
3. **Make Changes** - Update relevant documentation
4. **Test Links** - Ensure all internal links work
5. **Submit PR** - Create pull request with clear description

### Documentation Style Guide

#### Headers
```markdown
# Main Title (H1)
## Section (H2)
### Subsection (H3)
#### Detail (H4)
```

#### Code Blocks
```markdown
```typescript
// Always specify language
const example = "code";
```
```

#### Lists
```markdown
- Unordered list
  - Nested item
  
1. Ordered list
2. Second item
```

#### Links
```markdown
[Link Text](relative/path/to/file.md)
[External Link](https://example.com)
```
